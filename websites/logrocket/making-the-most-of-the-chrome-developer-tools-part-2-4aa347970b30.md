Most web developers are familiar with the basic features of the chrome developer tools‚Ää‚Äî‚Ääthe DOM inspector, styles panel, and JavaScript console‚Ää‚Äî‚Äähowever, there are a number of lesser-known features that can dramatically improve a debugging or creation workflow.

### Store as global¬†variable

![](https://storage.googleapis.com/blog-images-backup/1*vsnou_CyFAGBvcxX-Vit0g.png)

Inspecting complicated objects that are logged to the console can sometimes be tricky if they have many keys, or contain values that are hard to parse manually. Luckily, Chrome makes it easy to inspect such objects with JavaScript.

To do so, right click on an object in the console and press ‚Äústore as global variable‚Äù¬†. This stores the object as a global variable called `temp1` which you can then work with using JavaScript.

### Alt + Up / Alt +¬†Down

![](https://storage.googleapis.com/blog-images-backup/1*BN9Mrx_QN_va7uX9foSOww.png)

When debugging CSS, you can select a property and use the up/down keys to tweak it‚Äôs value. By default, the arrow keys adjust values by +/- `1`. However, by holding the `alt` key, you can use the arrow keys to adjust values finely in steps of `0.1`, which is particularly useful when working with fractional values.

Conversely, you can hold `shift` to adjust values in steps of `10`.

### Preserve Log

![](https://storage.googleapis.com/blog-images-backup/1*R9-tpjnSs1ssvTki90qlLw.png)

Preserve log is a checkbox that lets you persist logs between page refreshes. This is useful when debugging website issues that require you to refresh the page, since all console output is otherwise cleared. When this option is enabled, a new type of ‚ÄúNavigation‚Äù log appears in the console to show page refreshes or navigation events to different pages.

### Hold shift during selection mode

![](https://storage.googleapis.com/blog-images-backup/1*8NFF4gMKsGFhv-9HWK9-4g.png)

In [part one of this series](https://blog.logrocket.com/making-the-most-of-the-chrome-developer-tools-8cac9a206979#.hhqcm2lq7), I discussed selection mode, which is activated by pressing the icon in the upper-left corner of the DOM inspector. This lets you visually select elements on the page to inspect by hovering over them with your mouse.

I recently learned that by holding shift, you can go ‚Äúthrough‚Äù the topmost element in a stack, and select deeper elements. This is helpful for complicated sections of DOM, or a page with some type of overlay that covers other elements.

### Network + Log¬†Filters

![](https://storage.googleapis.com/blog-images-backup/1*F41OyOeFc3mdvcRGRHYedw.png)

When debugging an app that has a lot of network requests or console logs, it can be useful to filter for particular types of events. Chrome has a filtering language that supports many different properties, as well as operators like `*` to do wildcard matches.

If you type ‚Äú-‚Äù, Chrome will expose a typeahead that shows the various properties you can filter for. You can also toggle on ‚Äúregex‚Äù mode to do a regex match on the data shown in each row.

### Bonus: Dark¬†Theme

![](https://storage.googleapis.com/blog-images-backup/1*MO3U6DyiFUGfZrEaVfKUmw.png)

Chrome comes with a built-in dark theme for the dev tools. You can enable it by clicking the three dots icon in the upper right of the dev tools pane, clicking ‚Äúsettings‚Äù, and then toggling the theme.
